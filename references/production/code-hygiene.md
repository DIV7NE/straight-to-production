# Code & Project Hygiene — Anti-Garbage Rules

AI-generated code has a 90-100% rate of leaving garbage behind (OX Security, 300+ repos).
Read this BEFORE writing code. Follow these rules WITHOUT EXCEPTION.

## File Rules

### NEVER create these
- Duplicate utility files (one utils.ts, not utils.ts + helpers.ts + common.ts)
- Planning/analysis markdown files in the project root (plans go in PLAN.md, not scattered .md files)
- Backup/temp files (.bak, .old, .copy, .tmp)
- Empty placeholder files ("will implement later")
- Config files for tools not in use

### ALWAYS ensure these are gitignored
- Build output: .next/, dist/, build/, out/, target/
- Dependencies: node_modules/, .venv/, vendor/
- OS files: .DS_Store, Thumbs.db
- IDE files: .idea/, .vscode/ (unless team-shared settings)
- Environment: .env, .env.local (only .env.example committed)
- Cache: __pycache__/, .pytest_cache/, .mypy_cache/

## Comment Rules

### NEVER write these
- Comments restating what the code already says: `// increment counter` above `counter++`
- Step-by-step tutorial comments on every line
- "TODO: implement" without a plan to implement (if it's not in PLAN.md, delete it)
- Commented-out code blocks — delete it, git has the history
- AI attribution comments: "// Generated by Claude", "// AI-assisted"
- Obvious JSDoc/docstrings: `/** Returns the user */ function getUser()`

### ONLY write comments for
- WHY something is done a non-obvious way (not WHAT it does)
- Warnings about gotchas: "// Stripe requires raw body for webhook verification"
- Legal/license headers when actually required
- Links to relevant documentation or issue trackers

## Code Rules

### Before EVERY commit, verify
- Zero unused imports (linter catches these)
- Zero unused variables or functions
- Zero console.log / print / println in production code
- Zero empty catch blocks — catch must handle or rethrow
- Zero placeholder code ("throw new Error('Not implemented')")
- Zero hardcoded URLs, ports, or credentials
- Zero duplicate utility functions — search before creating
- Consistent naming convention throughout (whatever the project uses)

### Code smell limits
- No file over 300 lines — split it
- No function over 50 lines — extract helpers
- No more than 3 parameters per function — use an options object
- No nested callbacks deeper than 2 levels — use async/await or extract
- No barrel files (index.ts that re-exports everything) — import from specific paths

## Test Rules

### NEVER write these
- Tests with no real assertions: `expect(true).toBe(true)`
- Tests that test implementation: `expect(mockFn).toHaveBeenCalledTimes(3)` instead of testing the output
- Snapshot tests for dynamic content (snapshots are for stable UI only)
- Tests that depend on execution order
- Empty test files or describe blocks

### EVERY test must
- Test ONE behavior (not implementation details)
- Have a descriptive name: "returns error when invoice has no line items"
- Actually fail when the feature is broken (verify by breaking the code)
- Clean up after itself (no test pollution)

## Dependency Rules

- NEVER install a package without verifying it exists and is actively maintained
- NEVER have two packages that do the same thing (pick one, remove the other)
- NEVER leave devDependencies mixed with production dependencies
- Run dependency audit before every commit
- Remove packages immediately when no longer imported

## Git Rules

- NEVER commit binary files (images go in CDN/storage, not git)
- NEVER commit with vague messages ("update", "fix", "changes")
- NEVER leave merge conflict markers in code
- Every commit: one logical change, descriptive message
- Keep .gitignore comprehensive and up-to-date

## Post-Build Hygiene Check

After completing ANY feature, scan for:
1. Unused imports/variables: linter
2. Console/debug statements: `grep -rn "console\.\(log\|warn\|debug\)" --include="*.ts" --include="*.tsx"`
3. Commented-out code: `grep -rn "^\s*//.*[{(]" --include="*.ts"` (heuristic)
4. TODO/FIXME: `grep -rn "TODO\|FIXME\|HACK\|XXX" --include="*.ts"`
5. Empty catch blocks: `grep -rn "catch.*{" -A1 --include="*.ts"` (check for empty body)
6. God files: `find src -name "*.ts" -exec wc -l {} + | sort -rn | head -5` (flag 300+)
7. Duplicate code: run jscpd or similar duplicate detector
8. Unused dependencies: run depcheck / pip-check / cargo-udeps
